# Demo Brief — T-001 AI DEMO 孵化平台

> 状态：探索中，可持续讨论。全平台蓝图在此文件维护；当前批准的实现范围以 `generated/effective-spec.md` 为准。

## Background

发起人 Manta 是 sola「AI Agent 线上共学」二期学员（作品 #24 AUTO-Recruiter）。共学课程结束后，学员作品散落在微信群、网盘和各自的部署链接里，缺少一个持续孵化、展示和互相反馈的家。课程导师也建议学员手搓一个 AI DEMO 孵化平台。

参考对象：<https://sola.wamo.city/>（Social Layer「AI Agent 线上共学」三期官网）。实测其核心构成：

- **作品墙**：24 个学员 Demo，每张卡片 = 作者 + 作品名 + 一句话介绍 +（部分有）"打开 Demo" 链接。
- **7 条赛道**：紧急下班（自动化）、蒸馏（方法论）、产品、游乐园、内容创作、LifeOS、青少年创造。
- **社区 Agent**：截图排错、工具选型、项目陪跑、持续记忆。

sola 本质是课程平台，作品展示只是 Demo Day 的快照。本项目的定位是**项目全周期管理平台**，补上课程之外的长期环节：**领灵感 → 立项 → 推进度 → 被看见 → 收反馈**。灵感生成器不是附属玩具，而是全周期的入口：一个人从摇一摇领到 idea 开始做项目，后续的项目地图和进度管理（Phase 2）可以完美衔接前期的灵感领取（数据上通过 `origin_idea` 字段打通，见 effective-spec）。

## User

- **主用户**：共学学员 / 社区创作者——有项目想法或半成品，需要灵感、展示位和推进压力。
- **次用户**：导师与访客——浏览作品、试用 Demo、给建议。
- **现状**：创意靠灵感派对开放麦；进度靠微信群自觉汇报；展示靠 Demo Day 一次性收口；课后基本失联。

## AI Advantage

- **灵感生成器**：按赛道 × 人群 × AI 形态做结构化组合生成 idea（规则即可用，LLM 润色为可选增强），把「不知道做什么」变成「摇一摇就有起点」。
- **后续阶段**：项目陪跑/督促 Agent（对标 sola 社区 Agent 的"项目陪跑 + 持续记忆"），以及基于仓库数据的进度自动可视化。

## Roadmap（全平台分阶段蓝图）

### Phase 1 — 展示墙 + 灵感生成器（本次 MVP，见 effective-spec）

一个可公开访问的站点：Demo 作品墙 + 「摇一摇」灵感生成器。机器数据源 = 本仓库 `demos/*/case.yaml`；`demo-card.md` 仅供人阅读（**仓库即数据库**，不做双份维护）。无账号、无评论。

### Phase 2 — 项目进度管理与督促

每个项目的状态页（Idea → Building → Showcase 流转可视化，"小型可视化 GitHub"），进度时间线，进度停滞提醒等轻量督促机制。衔接 Phase 1 的灵感领取：从 `origin_idea` 链路可视化「从一次摇一摇到一个上墙作品」的完整旅程。

### Phase 3 — 试用与互评

Demo 在线试用入口、反馈/建议收集、轻量互评（先无账号的留言形式，后升级）。

### Phase 4 — 账号与社区化 + sola 数据管道合作

账号体系、个人项目管理页、社区运营机制。与 sola 的关系定位为**数据管道合作**而非合并：向 Shing 老师争取每期 Demo Day 的作品数据入口和新学员引流位（sola 每 22 天产出一批新作品，是平台最稳定的增量供给管道）。合并不作为目标；如果平台自己跑出留存数据，是否深度整合届时自然浮现（2026-07-21 圆桌共识：先独立验证，拿数据再上桌）。

## MVP Scope（Phase 1）

作品墙 + 摇一摇灵感生成器；执行细节（数据契约、词库规则、统计方案、技术栈、场景验收）以 `generated/effective-spec.md` 为准，不在此重复维护。

## Out of Scope

- Phase 1 不做：账号、评论、上传、在线试用、通知。
- 整体不做：支付、课程系统。
- 未经授权的他人作品不上墙（初期只收录本仓库孵化的 demo 与手动添加的授权条目）。

## 北极星指标与数据复盘

上线后跟踪三个指标（详见 effective-spec 的 Post-Launch Data Review）：

1. **灵感转化数**：多少个 demo 的 case.yaml 标注了 `origin_idea`——直接度量「灵感 → 项目」闭环是否真实发生。
2. **无催促的第二次提交数**：供给侧生死判据（来自公开 Git 历史）。
3. **回访与功能事件分布**：GoatCounter 聚合统计（访问量 + 摇一摇/复制/卡片点击等事件），仪表盘仅站点所有者可见；明确不采集停留时长（渐进方案取舍）。

上线约 30 天后产出完整数据复盘报告，**功能去留由用户定夺**，不设自动裁决。

## Risks（圆桌补记，2026-07-21）

- **"第 23 天问题"（四位思想家共识的最大风险）**：课程期活跃靠付费+导师+Demo Day 截止日三个外部激励，课一结束激励蒸发。Phase 2-4 全部建立在「课后社区持续活跃」这一未检验假设上，应视为假设清单而非承诺路线图。
- **激励结构提示（芒格）**：「导师建议做平台」是风险收益不对齐的结构——建议免费、失败成本全在发起人。导师热情不构成需求验证，以上线后的真实数据为准。
- **全周期定位的范围风险**：全周期是叙事和数据结构（`origin_idea` 成本≈0），不是 Phase 1 的功能范围；警惕它把 MVP 越做越大。
